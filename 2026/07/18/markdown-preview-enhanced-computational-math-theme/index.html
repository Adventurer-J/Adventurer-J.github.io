先确认适用范围

这份主题面向 VS Code 扩展 Markdown Preview Enhanced（下文简称 MPE），使用的是扩展提供的 style.less 自定义入口。它不通过 VS Code 内置预览的 markdown.styles 加载，也不应粘贴到 settings.json。

推荐选择工作区配置。样式只影响当前项目，便于随项目迁移，也不会改变其他 Markdown 仓库的预览外观。只有确定所有项目都使用同一排版时，才需要全局配置。

安装步骤

  1. 安装 Markdown Preview Enhanced

    在 VS Code 扩展面板中搜索 Markdown Preview Enhanced，确认扩展标识为 shd101wyy.markdown-preview-enhanced，完成安装后重新载入窗口。

  2. 打开工作区样式文件

    按 Ctrl + Shift + P；macOS 使用 Cmd + Shift + P。运行下面的命令：

    Markdown Preview Enhanced: Customize CSS (Workspace)

    扩展会打开当前工作区使用的 style.less。通过命令打开比手动猜测配置目录更稳妥。

  3. 覆盖 style.less

    先备份原内容，再把下载文件中的全部 Less 规则复制到 style.less，保存文件。不要在这里加入 JSON 设置；该文件只接受 CSS 或 Less。

  4. 打开并刷新增强预览

    打开 Markdown 文件，运行 Markdown Preview Enhanced: Open Preview to the Side。若预览已经打开，点击预览右上角的刷新按钮，让扩展重新把 Less 编译为 CSS。

功能说明

01 / TYPOGRAPHY

紧凑的论文式层级

正文采用中英文字体回退链，标题使用无衬线字体；收紧段落与标题间距，适合长篇数值实验记录。

02 / EQUATIONS

公式基线与间距修复

覆盖 KaTeX、MathJax 与 mjx-container，清除行内公式误套代码框的问题，并压缩独立公式上下留白。

03 / TABLES

数值结果三线表

表格整体居中、数字等宽，保留表头与上下边线，弱化内部网格，适合误差、收敛阶和耗时对比。

04 / CODE

代码与行内标识

行内代码保持紧凑，代码块使用浅底、细边与左侧强调线，长行可横向滚动。

05 / STRUCTURE

摘要与定理环境

内置摘要、定义、定理、引理、命题、推论、证明、注和警告等可复用结构。

06 / OUTPUT

打印与 PDF

打印时切换为黑白文字、缩小字号，并尽量避免公式、表格、图和定理环境跨页断裂。

公式与表格为何更适合计算报告

计算数学文档的视觉密度通常由公式和结果表决定。主题没有给公式添加卡片背景，而是保留论文式白底，并将独立公式的上下间距收紧为：

.katex-display,
.MathJax_Display,
.mathjax-exps,
mjx-container[display="true"] {
  margin: 0.34rem 0 0.46rem !important;
}

表格默认居中，最小宽度为正文的 72%，并启用 font-variant-numeric: tabular-nums，让相同位数的数据列更容易纵向比较。下面这类结果表尤其适合：

| N | Time (s) | errlinf_rel | errl2_rel | errh1_rel | errh2_rel |
|---:|---:|---:|---:|---:|---:|
| 32 | 0.18 | 2.31e-4 | 8.42e-5 | 1.73e-3 | 2.90e-2 |
| 64 | 0.71 | 5.78e-5 | 2.11e-5 | 8.66e-4 | 1.45e-2 |

使用摘要和定理环境

MPE 允许在 Markdown 中保留 HTML。给内容添加主题已经定义的类名，就能得到结构化的技术文档块：

<div class="abstract">
本文比较两类离散方法的误差、收敛阶与计算成本。
</div>

<div class="theorem">
在给定正则性条件下，离散解满足相应的误差估计。
</div>

<div class="proof">
由稳定性、一致性与三角不等式可得结论。
</div>
类名显示标签适用内容
.abstract摘要问题、方法与主要结论
.definition定义符号、空间与算子
.theorem / .lemma定理 / 引理理论结论与辅助结果
.proof证明推导过程，末尾自动显示证毕符号
.remark / .warning注 / 注意实现细节、边界条件与风险提示

按项目微调

建议把个性化覆盖写在 style.less 最后。后出现的规则优先级更高，升级主题时也更容易定位自己的修改。

导出 PDF

主题中的 @media print 会在导出阶段启用：页面去掉屏幕预览留白，字体切换为黑色，表格改为全宽，并为代码块、表格、图片和定理环境设置避免跨页。最终分页仍受纸张尺寸、浏览器打印引擎和内容高度影响，导出前应检查预览中的分页位置。

常见问题

现象检查方法
保存后没有变化确认打开的是 MPE 预览而非 VS Code 内置预览，然后点击预览右上角刷新按钮。
Less 编译报错检查是否把 settings.json 的 JSON 内容误贴进 style.less，并检查括号是否成对。
只有一个项目生效这是工作区配置的预期行为；如需全局生效，改用 Customize CSS (Global)。
公式仍带代码框确认最终文件完整复制，尤其不要遗漏清理公式内部 code 样式的规则。
表格过宽减少非必要列或缩短表头；宽表在屏幕上应优先保持数据完整，而不是强制压缩字号。
配置边界：这份 Less 只改变 MPE 的渲染结果，不改变 Markdown 源文件、VS Code 编辑器配色或其他 Markdown 扩展的预览。

命令名称、style.less 入口和刷新行为以 Markdown Preview Enhanced 的官方自定义 CSS 文档为准。
